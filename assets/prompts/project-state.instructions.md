# 项目状态文档规范

## docs/project-state.md 格式

```markdown
# {项目名} - 项目状态

## 基本信息
- **项目类型**: Web App / API / CLI / Mobile / Library
- **技术栈**: {具体技术}
- **创建日期**: {日期}
- **最后更新**: {日期时间}

## 当前阶段
{需求分析 / 架构设计 / 开发实现 / 测试验证 / 部署上线 / 运维}

## 里程碑
- [x] M1: {已完成里程碑}
- [ ] M2: {当前里程碑} ← 当前
- [ ] M3: {未来里程碑}

## 活跃任务
| ID | 标题 | 状态 | 指派 | 优先级 |
|----|------|------|------|--------|
| TASK-001 | ... | 进行中 | @va-developer | P0 |
| TASK-002 | ... | 待开始 | @va-tester | P1 |

## 已完成任务 (本阶段)
| ID | 标题 | 完成日期 |
|----|------|---------|
| TASK-000 | ... | {日期} |

## 已知问题
- [ ] {问题描述} - {影响范围}

## 技术债务
- [ ] {债务描述} - {优先级}

## 备注
{其他需要关注的信息}
```

## 更新规则

- 每次任务状态变更时更新
- 每次会话结束时由 @va-handover 更新
- 里程碑完成时更新
- 保持简洁，只记录当前活跃状态
