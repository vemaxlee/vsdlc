# SDLC INT (智能入口)

这是 `/sdlc-init` 的增强入口，用于“先需求、再类型、再技术栈”的启动方式。

## 流程要求

### 步骤1: 需求文档来源选择（对话框）
必须先使用 `vscode_askQuestions` 弹框，让用户在以下选项中选择，且必须包含 `稍后处理`:
1. `创建新的需求文档`
2. `使用已有需求文档`
3. `稍后处理`

如果选择“创建新的需求文档”:
- 在 `docs/requirements/REQ-001.md` 创建需求草案模板，并引导补充核心需求。

如果选择“使用已有需求文档”:
- 让用户选择或指定已有需求文档路径（优先 `docs/requirements/` 下文档）。
- 读取并提取关键需求要点，作为后续推断输入。

### 步骤2: 由 PM/Architect 推断项目类型，再由用户确认（对话框）
- 基于需求内容由 `@va-pm` 主导推断项目类型；如边界复杂或技术架构不明确，自动委派 `@va-architect` 辅助推断。
- 给出 1 个推荐类型 + 1-2 个备选类型及简短理由。
- 必须用 `vscode_askQuestions` 让用户最终确认，选项中必须包含 `稍后处理`。

### 步骤3: 推荐技术栈并确认（对话框）
- 基于已确认项目类型 + 需求约束给出推荐技术栈（推荐项 + 备选项）。
- 推荐内容至少覆盖：前端/后端/数据存储/测试/部署。
- 必须用 `vscode_askQuestions` 让用户确认，选项中必须包含 `稍后处理`。

### 步骤4: 后续步骤保持不变
- 从这里开始，完整复用 `/sdlc-init` 的后续流程（模板复制、变量替换、项目骨架生成、Git 初始化/提交等），不改变既有规则。
- 执行时使用步骤2/3确认后的项目类型和技术栈，不再重复询问。

## 执行约束
1. 优先使用模板复制脚本 `~/.copilot/bin/sdlc-template-init`
2. 按项目状态自动切换或委派必要代理/子代理
3. 遇到问题先自动修复，再委派；仅必须人工介入时弹对话框（含 `稍后处理`）
4. 任务结束、阶段完成或重大改动后及时提交改动
